# PRD: 단순 게시판 서비스

**버전:** 1.0  
**작성일:** 2026-04-29  
**상태:** Draft

---

## 1. 개요

누구나 글을 쓰고 읽을 수 있는 단순한 게시판 서비스다. 회원가입 없이 닉네임과 비밀번호만으로 글을 작성·수정·삭제할 수 있다.

---

## 2. 사용자 스토리

| ID | 역할 | 행동 | 가치 |
|---|---|---|---|
| US-01 | 방문자 | 게시글 목록을 본다 | 어떤 글이 있는지 파악한다 |
| US-02 | 방문자 | 게시글을 클릭해 내용을 읽는다 | 원하는 정보를 얻는다 |
| US-03 | 방문자 | 닉네임·비밀번호를 입력해 글을 쓴다 | 회원가입 없이 의견을 남긴다 |
| US-04 | 작성자 | 비밀번호를 입력해 글을 수정한다 | 내용을 바로잡는다 |
| US-05 | 작성자 | 비밀번호를 입력해 글을 삭제한다 | 원치 않는 글을 지운다 |
| US-06 | 방문자 | 게시글에 댓글을 단다 | 글에 반응을 남긴다 |
| US-07 | 방문자 | 키워드로 게시글을 검색한다 | 원하는 글을 빠르게 찾는다 |

## 3. 기능 명세

### 3.1 게시글 목록

- 최신순 정렬, 페이지당 20건
- 표시 항목: 번호, 제목, 닉네임, 작성일, 조회수, 댓글수
- 페이지네이션 (이전 / 다음)

### 3.2 게시글 상세

- 제목, 본문, 닉네임, 작성일, 조회수 표시
- 조회할 때마다 조회수 +1
- 하단에 댓글 목록 표시 (오래된 순 정렬)
- 삭제된 게시글 접근 시 "삭제된 게시글입니다" 안내 (API 응답 code: `POST_DELETED`)
- 존재하지 않는 게시글 접근 시 "존재하지 않는 게시글입니다" 안내 (API 응답 code: `POST_NOT_FOUND`)

### 3.3 게시글 작성

- 필수 입력: 닉네임, 비밀번호, 제목, 본문
- 닉네임: 2~20자
- 비밀번호: 4자 이상 (서버에서 해시 저장)
- 제목: 1~100자
- 본문: 1~5,000자

### 3.4 게시글 수정 / 삭제

- 수정·삭제 버튼 클릭 시 비밀번호 입력 모달 표시
- 모달 취소 버튼 클릭 시 모달 닫고 현재 페이지 유지
- 비밀번호 불일치 시 에러 메시지 노출
- 수정 가능 필드: 제목, 본문 (닉네임·비밀번호는 수정 불가)
- `/posts/:id/edit` 직접 URL 접근 시 비밀번호 미검증 상태이면 상세 페이지(`/posts/:id`)로 리다이렉트
- 삭제 시 소프트 삭제 (is_deleted 플래그), 해당 게시글의 댓글도 일괄 소프트 삭제 (cascade)

### 3.5 댓글

- 필수 입력: 닉네임, 비밀번호, 내용
- 닉네임: 2~20자
- 비밀번호: 4자 이상 (서버에서 해시 저장)
- 내용: 1~500자
- 댓글도 비밀번호로 삭제 가능
- 댓글 수정은 미지원 (1.0 범위 외)

### 3.6 검색

- 제목 + 본문 대상 키워드 검색
- 검색 결과도 최신순 정렬 + 페이지네이션 적용 (페이지당 20건)
- 빈 검색어(`?q=` 또는 `?q` 미전달) → 전체 목록 반환 (검색하지 않은 것과 동일)
- 게시글 0건 시 "아직 게시글이 없습니다" 안내 문구 표시 (목록·검색 결과 모두 적용)

## 4. 비기능 요구사항

| 항목 | 목표 |
|---|---|
| 응답 시간 | 목록·상세 페이지 로드 ≤ 1초 (p95) |
| 동시 사용자 | 최대 100명 동시 접속 처리 |
| 비밀번호 보안 | bcrypt 해시 저장, 평문 미보관 |
| 스팸 방지 | 동일 IP 분당 게시글 작성 5회 제한 (댓글 작성은 미적용) |
| 반응형 | 모바일(375px) / 데스크탑(1280px) 모두 지원 |

## 5. 데이터 모델 (요약)

```
post
  id            BIGINT PK
  title         VARCHAR(100)
  body          TEXT
  nickname      VARCHAR(20)
  password_hash VARCHAR(255)
  view_count    INT DEFAULT 0
  is_deleted    BOOLEAN DEFAULT false
  created_at    TIMESTAMP
  updated_at    TIMESTAMP

comment
  id            BIGINT PK
  post_id       BIGINT FK → post.id
  nickname      VARCHAR(20)
  password_hash VARCHAR(255)
  body          VARCHAR(500)
  is_deleted    BOOLEAN DEFAULT false
  created_at    TIMESTAMP
```

## 7. 범위 외 (Out of Scope, v1.0)

- 회원가입 / 로그인
- 파일·이미지 첨부
- 좋아요 / 추천
- 댓글 수정
- 관리자 페이지
- 알림 기능

---

## 8. 수락 기준 (Acceptance Criteria)

1. 닉네임·비밀번호·제목·본문을 입력하면 글이 목록에 나타난다.
2. 올바른 비밀번호 입력 시에만 수정·삭제가 완료된다.
3. 삭제된 글은 목록과 상세에서 보이지 않는다.
4. 키워드 검색 결과가 제목 또는 본문에 키워드를 포함한 글만 반환된다.
5. 모바일 화면(375px)에서 레이아웃이 깨지지 않는다.
